Mengatasi Custom Domain Github Pages Yang Error

March 19, 2024

Hai teman-teman! Saya mau berbagi pengalaman nih tentang proses nyambungin custom domain ke blog Hugo di GitHub Pages.

Ceritanya mulai dari yang seru-seru dulu ya!

Jadi kemarin, saya lagi semangat banget karena akhirnya berhasil membangun blog pakai Hugo SSG dan berhasil deploy-nya ke GitHub Pages. Terus, saya kepikiran, kenapa gak coba pake custom domain biar makin keren gitu.

Saya pun langsung deh sewa atau beli domain yang saya suka. Terus, saya ikuti tutorial yang ada dari provider domain dengan teliti banget dan akhirnya proses inipun selesai dengan baik hingga dapat diakses. Awalnya lancar, blog saya di load dengan baik ketika saya mengunjungi custom domain saya, tapi tiba-tiba pas saya membuat pembaharuan dan melakukan push update ke repo saya di GitHub, ehh malah pas cek domain custom-nya, muncul 404 dari GitHub yang bilang halaman gak ada! Kaget dong saya!

Pas saya cek setting GitHub Pages-nya, eh ternyata domain custom yang udah saya setting tadi kok ilang sendiri. Aneh banget kan? Ya udah, saya masukin lagi deh domain custom-nya ke setting GitHub Pages dan masalah pun selesai. Tapi, masalahnya terulang lagi setiap kali saya push ke repo-nya. Bikin frustasi banget asli!

Akhirnya, saya putusin buat cari tau penyebabnya sendiri. Dan ternyata masalahnya ada di file CNAME yang gak ada di projects blog saya di GitHub. Jadi ketika sudah saya buatkan file CNAME-nya, setiap saya melakukan push lagi ke repository, masalahnya gak terulang lagi dan gak ada lagi 404!

Nah, tipsnya simpel banget nih teman-teman, cukup masukin aja file CNAME yang berisikan domain custom-mu ke repo-nya di GitHub. Gampang kan?

Untuk kamu yang pakai teks editor untuk mengelola project, buat saja file dengan nama CNAME tanpa ekstensi dan di dalam file itu isi nama domain custom kalian tanpa http atau https. Namun jika kalian menggunakan terminal atau CLI untuk mengelola project, langsung saja pakai perintah echo atau nano. Jika menggunakan echo kalian cukup menulis perintah ini di root directory.

echo "namadomain.com" > CNAME Ini bakal membuat file baru dengan nama CNAME di folder tempat kamu jalanin perintah itu dalam case ini tadi saya bilang di root directory project hugo.

Jadi, buat kalian yang lagi mau nyoba juga, semoga pengalaman saya ini bisa jadi referensi ya ketika mendapati error yang sama. Sampai jumpa…
